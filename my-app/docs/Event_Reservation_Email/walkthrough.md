# イベント予約メール送信機能の実装

## 変更内容の概要
イベント予約（`Reservations` コレクション）登録時に、予約者（お客様）と管理者（はなとたね側）の双方に自動メールが送信されるように機能を実装しました。

## 実装の詳細
これまでコンソールへのログ出力（プレースホルダー）のみとなっていた `src/collections/Reservations.ts` の `afterChange` フックを更新し、以下の処理を追加しています。

- **`nodemailer` の設定**: サイトの他のフォームと同様に環境変数の SMTP 情報を読み込んでメール送信トランスポーターを構成しました。
- **イベント名の取得**: Payloadの `req.payload.findByID` を用いて、予約に紐づいたイベント（`column`）のデータを取得し、メール内でイベントタイトルを表示できるようにしました。
- **2種類のメール送信**:
  1. **お客様向け**: 予約完了をお知らせし、入力内容の控えを送信。
  2. **管理者向け**: 新規予約が発生したことを通知し、予約内容を送信（送信先は `CONTACT_EMAIL_TO`）。

## 確認手順
1. 本番環境またはローカル開発環境で、実際にイベント予約フォームにテスト投稿する。
2. フォームに入力したメールアドレス宛に「【NPO法人はなとたね】イベントのご予約を受け付けました」というメールが届くことを確認する。
3. `CONTACT_EMAIL_TO` に設定されているメールアドレス宛に「【イベント予約】新規のご予約がありました」というメールが届き、予約内容が記載されていることを確認する。
