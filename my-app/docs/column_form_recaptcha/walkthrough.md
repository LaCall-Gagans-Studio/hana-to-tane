# Google reCAPTCHA v3（Enterprise）への変更完了

ご提示いただいたGoogleの仕様（ボタン押下時に背後でトークンを取得する Enterprise/v3スコアベースの方式）に合わせて実装を完了しました。

## 変更内容
1. **パッケージの入れ替え**
   - 先行して入れた `react-google-recaptcha` をアンインストールしました。
   - 代わりに、Reactでv3方式（ユーザーに見えない状態でのトークン取得）を扱える公式ライブラリ `react-google-recaptcha-v3` をインストールしました。
2. **フロントエンドの実装 (`ReservationForm.tsx`)**
   - 画面にあった「私はロボットではありません」の**チェックボックスを撤去**し、元のデザインに戻しました。
   - フォーム全体を `GoogleReCaptchaProvider` で包み、送信ボタンが押されたタイミング（`handleSubmit`の最初）で自動的に `executeRecaptcha` 関数を呼び出し、バックグラウンドでGoogleからトークンを取得してバックエンドへ送るようにしました。
3. **バックエンドの判定実装 (`actions/reservation.ts`)**
   - 従来通りGoogleのAPIでトークンを検証します。
   - 今回はv3仕様のため、検証結果に人間らしさの「スコア (`score` :0.0〜1.0)」が付与されます。
   - **スコアが 0.5 未満** の場合はスパムと断定し、予約情報の保存をブロックする仕組みを追加しました。

## 本番への導入時の操作
今回の実装により、ユーザーはチェックボックスを押す手間がなくなり、完全にシームレスな入力体験となります。

> [!WARNING]
> スコアの基準（0.5）について
> 今回のスパム判定閾値は一般的な `0.5` としています。しかし本稼働後に「普通に入力しているのにスパム扱いされて送信できない」というお客様がいらっしゃった場合は、少し緩くして `0.3` 程度に下げる調整をおすすめします。調整箇所は `actions/reservation.ts` の `recaptchaData.score < 0.5` の部分です。

以上でreCAPTCHA v3 スコアベースへの移行は全て完了となります！ローカルサーバー等で動作をご確認ください。
