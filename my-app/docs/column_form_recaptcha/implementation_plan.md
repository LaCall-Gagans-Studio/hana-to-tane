# Google reCAPTCHA v3（Enterprise スコアベース）への変更計画

ユーザー様がGoogle Cloudコンソール上に表示された手順（「ボタン押下時… grecaptcha.executeを呼び出します」）を提示されたため、
先ほどの「チェックボックスを表示するv2方式」から**「ユーザーに見えないところで自動でリスク判定を行う v3 / Enterprise 方式」**に実装方針を切り替えます。

## v3 スコアベース方式での実装計画

### 1. 使用パッケージの変更
- 追加した v2用パッケージ（`react-google-recaptcha`）は削除します。
- 代わりに React/Next.js 環境で v3（grecaptcha.executeの実行）を簡単に扱える `react-google-recaptcha-v3` をインストールします。

### 2. フロントエンドの実装変更 (`ReservationForm.tsx`)
- UI上のチェックボックス（`<ReCAPTCHA>`コンポーネント）は**撤去**します。これによって元のデザインを崩さずに済みます。
- `GoogleReCaptchaProvider` でフォームをラップし、`useGoogleReCaptcha` フックを利用して `executeRecaptcha` 関数を呼び出せるようにします。
- **「ボタン押下時の処理 (handleSubmit)」の中で**：
  1. `executeRecaptcha('reservation_submit')` を呼び出し、背後でトークンを取得します（ユーザーの指示通りの挙動です）。
  2. 取得したトークンをサーバー・アクションに送ります。

### 3. バックエンドの判定処理変更 (`actions/reservation.ts`)
- トークンの検証APIエンドポイント自体は同じ設定で動きます。
- ただし、v3の場合は「成功(success)」に加えて**「スコア(score: 0.0〜1.0)」**が返却されます。
- ロボットではなく人間らしいかの判定基準として、例えば「**スコアが 0.5 以下ならスパムとみなして弾く**」というロジックを追加します。

---

## User Review Required

> [!IMPORTANT]
> **スコアの基準について**
> 自動で人間かボットかを判定する「スコア(0.0〜1.0)」は、通常0.5を閾値にしますが、本番運用してみて厳しすぎる（普通のお客さんがエラーになってしまう）場合は 0.3 などに下げる調整が必要になる場合があります。  
> 初回は一般的な **0.5** を基準(0.5以上のスコアで許可)として実装いたします。
